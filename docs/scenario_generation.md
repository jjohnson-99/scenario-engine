# Monte Carlo Scenario Generation — Pipeline Explanation

## The calibration–generation pipeline

```
TimeSeries (historical data)
      ↓
[1] Calibration
      ↓
StochasticForecaster::calibrate(series)
      ↓
[2] Scenario Generation
      ↓
ScenarioGenerator::generate(model, history, n_scenarios, horizon_steps)
      ↓
vector<TimeSeries>  ← n_scenarios simulated future paths
```

---

## Step 1 — Calibration: where does `variance_` come from?

A deterministic forecaster like `MovingAverageForecaster` always returns `ForecastResult::variance = 0.0` — it
has no notion of uncertainty. A stochastic forecaster needs to *learn* how noisy its predictions are from the
historical data. That is what `calibrate()` does.

`variance_` is a **private member variable** (a field that lives inside the forecaster object). It is
declared in the header as `double variance_{0.0}`, which is *default member initialization* — the value is
`0.0` the instant the object is constructed. Calling `calibrate()` overwrites it:

```
StochasticMovingAverageForecaster::calibrate(series)
    │
    ├─ ResidualGenerator::generate(model, series)
    │       Walk-forward backtest:
    │       for each time t from minimum_observations() to series.size():
    │           history  = series.first_n_view(t)          // what the model would have seen
    │           predicted = model.forecast(history).mean    // what the model would have predicted
    │           actual    = series.value_at(t)              // what actually happened
    │           error     = actual - predicted
    │       → ResidualSeries{all (actual, predicted, error) triples}
    │
    └─ ResidualAnalyzer::analyze(residualSeries)
            variance = E[e²] − E[e]²   (single-pass population variance of the errors)
            → ResidualStatistics{mean_error, variance, standard_deviation}

variance_ = stats.variance   ← written into the forecaster object
```

After calibration, `forecast()` returns `ForecastResult{.mean = MA_prediction, .variance = variance_, ...}`.
The variance is a learned parameter — it describes how spread out the model's historical errors were.

## Step 2 — Monte Carlo Scenario Generation

With a calibrated forecaster, we can simulate many plausible futures. Each scenario is a separate path
generated by a loop that:

1. forecasts the next value (getting a `mean` and a `variance`),
2. samples a random shock from N(0, √variance),
3. appends the noisy next value to the growing history,
4. repeats for each future step.

```
ScenarioGenerator::generate(model, history, n_scenarios, horizon_steps)

    last_date = history.observations().back().timestamp

    for s = 1 … n_scenarios:
        path = copy of history          // each scenario starts from the same history

        for step = 1 … horizon_steps:
            result = model.forecast(path.view())
            │   result.mean     = the deterministic prediction (e.g., moving average)
            │   result.variance = learned noise level (from calibration)
            │
            noise ~ N(0, √result.variance)     ← random shock sampled here
            next_value = result.mean + noise
            │
            path.add(Observation{last_date + step days, next_value})

        store completed path
    end

    return vector<TimeSeries>  ← the scenario fan
```

This loop is **recursive**: each simulated value is appended to the growing history before the next
forecast is made, so earlier simulated values influence later ones. This is the right approach for
*projecting* into an unknown future. It differs from the **direct-horizon** mode used in the `Backtester`,
which evaluates a model against *known* history without feeding predictions back in.

## The new C++ feature: `<random>`

Sampling noise introduces three types from `<random>`:

| Type | Role |
|---|---|
| `std::random_device` | Hardware entropy source — used once to produce a seed |
| `std::mt19937` | Pseudo-random engine (Mersenne Twister) — fast, statistically strong |
| `std::normal_distribution<double>` | Maps uniform RNG output to N(μ, σ) samples |

```cpp
// Construction (once, in ScenarioGenerator constructor)
std::mt19937 rng{std::random_device{}()};   // seed from hardware entropy

// Usage (each step of each scenario)
std::normal_distribution<double> noise{0.0, std::sqrt(result.variance)};
double shock = noise(rng);                  // draws one sample
```

A **fixed seed** (`ScenarioGenerator sg{42}`) makes scenarios fully reproducible — every run produces the
exact same fan of paths. A `random_device` seed gives non-deterministic production runs.

When `variance == 0.0` (a deterministic model), `σ = 0` and no noise is added — all scenarios collapse
to the same path. This is correct behaviour: a model that claims zero uncertainty produces no spread.

---

## What the output looks like

`generate()` returns `std::vector<TimeSeries>`. Each element is a complete `TimeSeries` containing the
original historical observations **plus** the `horizon_steps` simulated future values. The scenarios can
be inspected to compute:

- Expected terminal value (mean of the last value across all scenarios)
- Spread / uncertainty (min, max, or a percentile band of the last value)
- Full path distributions — input to the optimizer in Phase 4
